# 随机数据特别处理

关键：**别把随机差异当作者错误**。

## 问题背景

当数据通过代码随机生成（如仿真研究、合成数据集），论文中的表格往往来自某一次特定运行，而复现时重新运行会产生不同的随机数据。

即使使用相同的随机种子，不同版本的库（如 numpy 1.x vs 2.x）也可能产生不同的随机数序列。

## 处理方法

### 1. 多种子重跑
用**多个随机种子（如 200 个）**重跑代码，统计每个指标的取值范围。

**示例**：
```python
results = []
for seed in range(200):
    np.random.seed(seed)
    X, y = generate_data()  # 随机生成数据
    model.fit(X, y)
    acc = model.score(X_test, y_test)
    results.append(acc)

# 统计范围
print(f"Accuracy range: [{min(results):.4f}, {max(results):.4f}]")
print(f"Mean: {np.mean(results):.4f}, Std: {np.std(results):.4f}")
```

### 2. 分类判断

将论文值与多种子范围比对，分两类：

#### （1）随机实现内
**定义**：论文值在多种子范围内。

**判断**：非作者错误，属于正常随机波动。

**注意**：
- 如果数据生成代码中效应大小是写死的（如 `effect = 0.5`），则"对上"只是循环验证，无实证意义。
- 需在报告中说明："数据随机生成，论文值在合理范围内，但效应大小预设，非数据驱动发现。"

#### （2）结构性不可能
**定义**：论文值超出任何种子的范围，或与数据生成结构必然关系矛盾。

**示例**：
```python
# 数据生成结构
pre = np.random.randn(100)
effect = 0.5
post = pre + effect + np.random.randn(100) * 0.1

# 必然关系：回归 post ~ pre，系数必 ≈ 1
# 如果论文报告系数 = 0.3，则结构性不可能
```

**判断**：真问题，任何种子都产不出论文值。

### 3. 检查生成结构的必然关系

**常见必然关系**：
- `post = pre + effect` → 回归系数 pre ≈ 1
- 独立生成的两个变量 → 回归系数 ≈ 0 且不显著
- `y = X @ beta + noise` → 回归系数必接近预设的 beta

**判断**：论文若与必然关系矛盾，即结构性不可能。

## 报告中的呈现

在复现指标对比表中增加"多种子范围"列：

| 指标 | 论文值 | 实跑值 | 多种子范围 | 结论 |
|------|--------|--------|-----------|------|
| 回归系数 (pre) | 0.98 | 1.02 | [0.95, 1.05] | 随机实现内，接近 |
| 回归系数 (independent) | 0.35 | 0.01 | [-0.05, 0.05] | **结构性不可能** |

## 阻断性问题判定

- **随机实现内**：非阻断性问题，不影响复现率
- **结构性不可能**：阻断性问题，需在问题反馈中说明
