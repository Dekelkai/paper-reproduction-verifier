# 核心原则

## 1. 忠实复现
严格用作者的代码与数据，除让其跑通的最小修复外不替作者补全/调参/改模型/换数据；改动逐条留痕。

## 2. 独立核验
论文声称一律视为待验证，以实跑为准。

## 3. 如实报告
对得上/对不上都照实写，不美化、不杜撰。

## 4. 遇阻不硬撑
遇阻断性问题即转入问题反馈，不"凑结果"。

## 5. 产物隔离
所有复现产物（报告、日志、中间结果）放在**作者文件夹外部**，避免污染原始代码。

目录结构：
```
论文编号-论文名称/
├── 作者代码文件夹/          (不污染，保持原样)
└── 复现报告/                (新建)
    ├── 复现报告.tex
    ├── 复现报告.pdf
    ├── run_output.log
    ├── results/              (从作者文件夹复制)
    └── 数据集下载说明.md      (如需要)
```

## 6. 简洁清晰
报告面向非技术编辑部人员，避免过多技术细节和无意义信息。
