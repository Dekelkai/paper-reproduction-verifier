# 报告撰写规范

## 目标读者

**非技术编辑部人员**：需要简洁清晰的结论，不需要过多技术细节。

## 核心原则

1. **简洁清晰**：避免冗余信息和技术术语堆砌
2. **聚焦阻断性问题**：问题反馈只写导致无法复现的问题
3. **保持模板结构**：使用 `templates/复现报告模板.tex`
4. **证据可追溯**：关键结论给出依据（但不过度展示技术细节）

---

## 各节撰写要点

### 1. 基本信息

**随稿文件**：
- ✅ 只列作者提供的文件（论文、代码、README、Dockerfile、requirements.txt 等）
- ❌ 不包含用户自己准备的文件（如 kaggle.json API 凭证）

**示例**：
```latex
随稿文件 & 论文 docx (3.9MB) / 代码 main.py (1103行) / Dockerfile / requirements.txt / README.md / 数据集需通过 Kaggle API 下载 (GTZAN 1.2GB) \\
```

### 2. 复现环境

**只写关键信息**：
- 操作系统（简要）
- Python 版本
- 核心依赖（5-8个关键包及版本）
- 数据集
- 运行时间

**删除无意义信息**：
- ❌ GPU 可用但未使用
- ❌ 某依赖未安装但不影响核心流程
- ❌ 过于详细的版本号列表（matplotlib, seaborn 等非关键包）
- ❌ 详细的硬件配置（CPU 型号、内存大小等，除非相关）

**示例**：
```latex
操作系统      & Windows 11 \\
Python 版本   & Python 3.10.20 \\
关键依赖      & numpy 1.26.4, pandas 2.2.3, autogluon 1.2.0, xgboost 3.1.3, lightgbm 4.6.0 \\
数据集        & GTZAN (Kaggle, 1.2GB, 9990 样本) \\
运行时间      & 约 15 分钟 \\
```

### 3. 复现过程记录

**代码与论文一致性分析**：
- 用通俗语言概括（"核心模型真实训练"，不需要列代码行号）
- 面向编辑部人员，不需要过多技术细节

**示例**：
```latex
\begin{enumerate}
  \item \textbf{核心模型真实训练}：AutoGluon 和基线模型均真实执行训练
  \item \textbf{数据集真实}：使用 Kaggle 公开数据集，非合成数据
  \item \textbf{指标真实计算}：所有论文指标均由代码真实计算
  \item \textbf{结果非硬编码}：论文数值未在代码中硬编码
\end{enumerate}
```

### 4. 复现指标对比

**清晰呈现**：
- 用表格逐项对比
- 标注差异是否在合理范围内
- 给出简要归因（随机种子、数据划分等）

**差异说明**：
- 放在表格后，用列表简要说明
- 不需要展开技术细节

### 5. 问题反馈（核心）

**仅记录阻断性问题**。

#### 阻断性问题定义

导致**无法复现**或**复现率显著降低**的问题：
- 核心模型无法训练
- 数据集无法获取
- 关键指标无法计算
- 代码无法运行（环境/依赖致命错误）
- 结果与论文差异 **>5%**
- 数据完全合成且无实证价值
- 结果硬编码

#### 非阻断性问题（不记录）

- 轻微数值差异（<2%，在随机误差范围内）
- AutoML 模型选择差异（正常行为）
- 可选模块缺失但不影响核心结果（如 TensorFlow/MLP 可选）
- 建议性改进（文档完善、代码规范等）
- GPU 未使用但不影响运行
- 依赖版本略有差异但不影响结果

#### 撰写格式

**如果无阻断性问题**：
```latex
\section{问题反馈}
\noindent 本次复现未发现阻断性问题。代码可直接运行，核心结果高度一致。
```

**如果有阻断性问题**：
```latex
\section{问题反馈}

\issue{数据集问题}{数据集无法获取，作者未提供下载链接，Kaggle 数据集已删除。}{README.md 提到需下载 XYZ 数据集，但链接失效。}{建议作者提供数据集镜像或替代下载方式。}

\issue{代码问题}{核心模型未实现训练，仅初始化模型但从未调用 fit() 方法。}{main.py:150-200 仅初始化模型，无训练循环。}{建议作者补充完整的训练代码。}
```

### 6. 结论

**分析**：
- 从"工程运行"与"论文复现"两个角度简要分析
- 列出可复现的部分与阻断性问题

**判定**：
- 完全复现 / 部分复现 / 无法复现
- 一句话理由

**复现率**：
- 给出百分比（如 99%）
- 简要说明扣分原因（如扣1%因AutoML模型组合略有差异）

## 行文风格

### ✅ 推荐风格
- 简洁：一句话说清楚
- 通俗：避免技术术语
- 结论导向：先给结论，再给依据

### ❌ 避免风格
- 冗长：过多技术细节
- 术语堆砌：`main.py:422 TabularPredictor.fit()`
- 过程导向：描述操作过程而非结果

### 示例对比

**❌ 冗余写法**：
```
GPU/CPU 配置: GPU: NVIDIA GeForce RTX 4060 Ti 16GB（可用但未使用）；
              CPU: 多核心运行；
              依赖未安装 tensorflow（requirements.txt 明确指定 
              tensorflow-cpu==2.15.1 但未安装，不影响核心流程）
```

**✅ 精简写法**：
```
关键依赖: numpy 1.26.4, pandas 2.2.3, autogluon 1.2.0, xgboost 3.1.3
```

---

**❌ 技术细节过多**：
```
核心模型真实训练：AutoGluon TabularPredictor.fit() (main.py:422)、
基线模型 cross_val_predict() + .fit() (main.py:467-535) 均真实执行
```

**✅ 通俗概括**：
```
核心模型真实训练：AutoGluon 和基线模型均真实执行训练
```

---

**❌ 非阻断性问题误写入问题反馈**：
```
【依赖问题】TensorFlow 未安装导致 MLP 被跳过，但不影响核心结果
【模型组合差异】AutoGluon 选择的模型与论文略有不同，这是正常行为
```

**✅ 正确做法（无阻断性问题）**：
```
本次复现未发现阻断性问题。代码可直接运行，核心结果高度一致。
```
